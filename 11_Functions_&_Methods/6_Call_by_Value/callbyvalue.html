Java is Always Call by Value

Understanding how Java passes arguments to methods.

The Golden Rule

When you pass a variable to a method, the method receives a copy of the variable's value, not the original variable itself.

1. For Primitive Types

For types like `int`, `double`, `char`, etc., the actual data value is copied. Changes made inside the method do not affect the original variable.

2. For Objects

The value of the reference (the memory address) is copied. This means the method's copy points to the same original object.

Example: The `Person` Object

The Code:

class Person {
    String name;
}

public class Main {
    public static void main(String[] args) {
        Person p1 = new Person();
        p1.name = "Abhay";

        // Try to change the name
        changeName(p1);
        System.out.println("After changeName: " + p1.name);

        // Try to assign a new object
        newPerson(p1);
        System.out.println("After newPerson: " + p1.name);
    }

    static void changeName(Person p) {
        // This changes the original object's data
        p.name = "Rudra";
    }

    static void newPerson(Person p) {
        // This points the local copy to a new object
        p = new Person();
        p.name = "Singh";
    }
}

Step-by-Step Story:

1

A `Person` object is created. Its `name` is "Abhay". The variable `p1` holds its memory address.

2

In `changeName(p1)`, the method gets a copy of the address. It uses this to find the original object and change its name to "Rudra". This works!

3

In `newPerson(p1)`, the method gets another copy of the address. But then it reassigns its local copy to a brand new object. The original `p1` reference outside is untouched.

Final Output

After changeName: Rudra

After newPerson: Rudra

Key Takeaway

Change Object's Content?
✅ Yes, the change is seen outside.

Point to a New Object?
❌ No, the change is NOT seen outside.
