Inbuilt vs. User-Defined Methods

A Chef's Guide to Java Methods

The Story: Chef Java's Kitchen

Imagine you are a programmer, but let's call you "Chef Java". Your kitchen (your program) needs to perform many tasks. For some tasks, you use standard, pre-built kitchen appliances. For others, you create your own special recipes from scratch. These are the two types of methods in Java!

Inbuilt Methods

(The Kitchen Appliances)

These are ready-made methods that come with Java. You don't need to know how they work internally, just what they do and how to use them. They are like a microwave or a blender â€“ reliable tools for common tasks.

Example: `Math.max()`

A built-in tool to find the larger of two numbers.

// Using a pre-built tool
int largerNumber = Math.max(10, 20);
System.out.println(largerNumber); // Output: 20

User-Defined Methods

(The Chef's Special Recipes)

These are methods you, the programmer, create for a specific, custom task. They are your own unique recipes to make your code organized, readable, and reusable.

Example: `greetChef()`

A custom recipe to print a special greeting.

// Writing our own recipe
public static void greetChef() {
    System.out.println("Hello, Chef Java!");
}
// Now we can use our recipe
greetChef(); // Output: Hello, Chef Java!

Chef Java's Full Program

Here's how the chef uses both an appliance and a special recipe in the kitchen.

public class Kitchen {

    // This is our special recipe (User-Defined Method)
    public static void makeSignatureDish() {
        System.out.println("Preparing the special pasta...");
    }

    public static void main(String[] args) {
        // Using our own recipe
        makeSignatureDish();

        // Using a built-in appliance (Inbuilt Method)
        double temperature = Math.sqrt(144.0);
        System.out.println("Oven temperature is set to: " + temperature);
    }
}

Output:

Preparing the special pasta...
Oven temperature is set to: 12.0
